# 1110. Delete Nodes And Return Forest

---

## ğŸ“Œ Problem Statement

You are given:

* The **root of a binary tree**
* An integer array `to_delete`

Delete all nodes whose values appear in `to_delete`.
After deletions, return the **forest** (list of tree roots) that remains.

---

## ğŸ” Key Insight

* When a node is **deleted**, its **non-null children** become **new roots**.
* The original root should be added to the result **only if it is not deleted**.
* Tree structure must be maintained for undeleted nodes.

---

## ğŸ§  Approach: Postorder DFS (Bottom-Up)

### Why Postorder?

* Children must be processed **before** deciding whether to delete the parent.
* Helps correctly reconnect or detach subtrees.

---

## ğŸ›  Algorithm Steps

1. Store `to_delete` values in a **HashSet** for O(1) lookup
2. Traverse the tree using **postorder DFS**
3. If current node is in `to_delete`:

   * Add its non-null children to result
   * Return `null` to parent (delete node)
4. Otherwise:

   * Return the node
5. After traversal:

   * If root is not deleted, add it to the result

---

## âœ… Correct & Best Solution

```java
class Solution {
    public List<TreeNode> delNodes(TreeNode root, int[] to_delete) {

        List<TreeNode> result = new ArrayList<>();
        Set<Integer> set = new HashSet<>();

        for (int val : to_delete) {
            set.add(val);
        }

        root = deleteHelper(root, result, set);

        if (root != null) {
            result.add(root);
        }

        return result;
    }

    private TreeNode deleteHelper(TreeNode root, List<TreeNode> result, Set<Integer> set) {
        if (root == null) return null;

        root.left = deleteHelper(root.left, result, set);
        root.right = deleteHelper(root.right, result, set);

        if (set.contains(root.val)) {
            if (root.left != null) result.add(root.left);
            if (root.right != null) result.add(root.right);
            return null; // delete current node
        }

        return root;
    }
}
```
